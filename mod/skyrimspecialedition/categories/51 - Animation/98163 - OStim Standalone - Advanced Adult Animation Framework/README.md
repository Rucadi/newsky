**OStim Standalone** is the continuation of OStim, adding and reimplementing features that developers and users alike always wanted to have.
  
Unlike OStim and OStim NG the Standalone version does **not** require OSA since it comes with its own UI, hence the name.
  
Being compiled in *C-Lib NG*, OStim Standalone works on every version of *Skyrim: Special Edition*.
  
  
*List of some new features*:
  
- Sex toy integration
  
- Furniture support
  
- Tongue support
  
- Undressing and redressing done in C++
  
- Heels scaling done in C++
  
- SoS bending
  
- random animation selection for OStim addons
  
- an alignment menu
  
- a search menu
  
- conditional navigations (MM scenes won't show navigation to vaginal sex animations, etc)
  
- a scene definition in .json instead of .xml
  
- custom facial expressions via .json
  
- custom furniture types via .json
  
- custom voice sets via .json
  
- the ability to use custom icons in animation packs

First of all, what is OStim?
  
 
  
*OStim* is an animation framework with **navigation** and a **stimulation system**.
  
  
Basically, OStim allows you to start **scenes** with NPCs and play various kinds of animations.
  
The **navigation menu** allows you to transition between idle animations that branch into **romantic** or **sexual animations**.
  
The **stimulation system** gives characters a new stat during animations: "Excitement". Excitement measures how close a character is to orgasm. This stat rises and falls over the course of a scene depending on what you are doing in it, in a natural way.
  
  
  
Full-auto Mode
  
  
You can optionally enable a full auto mode and let the scenes play out in front of you, hands free.
  
The scenes aren't just randomly selected, OStim will try to choose scenes that fit the environment.
  
  
  
QOL Features
  
  
Within the OStim Standalone MCM there are many QOL options you can tweak.
  
You can prevent certain clothing slots from being undressed.
  
You can tweak your excitement rate multiplier.
  
You can hide the excitement bars, among other things.
  
  
You can also configure OStim to enable first person automatically at scene start.
  
  
OStim Standalone is compatible with controllers, just map your keys correctly.
  
  
First Person
  
If you want to see the animations in first-person perspective, get [Improved Camera](https://www.nexusmods.com/skyrimspecialedition/mods/93962).
  
The OStim Standalone main download includes a profile for Improved Camera.
  
You can select the profile in game by pressing *Left Shift + Home* (the menu requires ReShade to be installed in order to appear).
  
  
  
How is OStim different than SexLab or Flower Girls?
  
  
OStim Standalone is more than just an animation player. In OStim you have control over the scene. You can choose what the actors do next via the UI which lets you navigate over the animation graph. And even in auto mode the next animation is chosen completely random, so you'll never see the same exact scene twice. (well.. technically if you play many many scenes at some point you probably will, but by that time you've already forgotten that you've seen this scenario already)
  
  
SexLab does not contain UI or navigation. It always plays the same sequence of animations. After a while whenever a scene starts you will already know exactly what's going to happen next just by seeing the first animation.
  
  
FlowerGirls is closed permissions and unmaintained. In other words, it's a **Dead Mod**.
  
  
It should be noted, however, that you can run OStim Standalone, SexLab and Flower Girls all at the same time without issues. So you don't have to miss out on any of the SL exclusive content if you want to use OStim!
  
  
  
Add-ons and compatibility
  
  
OStim Standalone has an active community of developers working on various add-ons.
  
It should go without saying that mods that use events from other animation frameworks like SexLab or Flower Girls will not receive events from this mod.
  
However, those mods do not touch OStim Standalone and vice-versa, so you may use them at the same time.

**Installing**
  
  
Follow the [installation guide](https://www.nexusmods.com/skyrimspecialedition/articles/5654). **When updating from OStim NG 6.8 or earlier a clean save is required!**
  
 
  
**Uninstalling**
  
  
Run these in console:
  
*stopquest OsexIntegrationMainQuest*
  
and
  
*stopquest OsexIntegrationSecondBarQuest*
  
  
Save, then remove the mod.
  
Then, reload the save and save again.
  
Using Fallrim Tools' Resaver to clean unused scripts is also recommended.
  
  
**Upgrading**
  
  
Some updates may require a "**clean save**" procedure.
  
1. Disable OStim, and all OStim add ons in your mod manager (including iWant Widgets if you have ORomance). Disabling animation packs is not necessary.
  
2. Launch the game and it will complain about missing mods.
  
3. Play the game for a minute or two. Run around, change cells, etc.
  
4. Save the game.
  
5. Open that new save in Fallrim Tools ReSaver﻿ and clean it (refer to ReSaver's mod page description to understand how to do this). <https://www.nexusmods.com/skyrimspecialedition/mods/5031>
  
6. Re-enable OStim and any other related plugins that were disabled in step 1. If you are upgrading, install the new OStim/addon update.
  
8. Launch game.
  
9. Load the clean save you made.
  
10. Give the game a few minutes to get the OStim MCM working. The game now believes you have never had OStim installed because you cleaned your save.
