**Campfire & Frostfall
  
-
  
Unofficial SSE Update**

  
  
**Description:**
  
This package contains all files needed to obtain a full functional Campfire & Frostfall mod.
  
It brings back Interface widgets, display about warmth & rain protection and a full working MCM menu.
  
  
**It is not the old trick which consist to use an empty skse esp. It uses updated script by Chesko himself**.
  
  
These files are downloadable from his github.
  
  
This pack contains files extracted from Campfire & frostfall LE and all needed scripts to make Campfire & Frostfall works with last SKUI version.
  
I decided to release this mod for Campfire & Frostfall nexus users convenience.
  
  
**Important information:**
  
You must do it on a new game or in a game where you didn't start Frostfall mod yet.
  
  
**Instruction for Campfire 1.12.1 SE & Frostfall 3.4.1SE****How to install:**
  

  
1. Install original [Campfire 1.12.1 SE](https://www.nexusmods.com/skyrimspecialedition/mods/667)﻿ & [Frostfall 3.4.1SE](https://www.nexusmods.com/skyrimspecialedition/mods/671)﻿
Install this mod, overwrite when asked
  
2. Install [PapyrusUtil SE](https://www.nexusmods.com/skyrimspecialedition/mods/13048)﻿, overwrite if asked
Launch Skyrim SE
  
3. Start Frostfall by MCM
Select a widget layout (if you don't do that, they are stuck in your up left screen corner)
  
4. Enjoy !

How to update from 1.11SE to 1.12SE:
  

  
1. Follow Step 1 to 3 overwrite when asked.
Install this mod (if you use mod manager, you must replace my mod by the new version)
  

  
It is not mandatory but I strongly suggest you to use [SSE Engine Fixes (skse64 plugin)](https://www.nexusmods.com/skyrimspecialedition/mods/17230)﻿**Knows issues:**
  
(testers report, I never faced this problem)
  
If you buy/sell **a lot of** armors / clothes **very quickly**, game may crash
  
If you break down a **big amount** of armors / clothes **very quickly**, game may crash
  
  
**It shouldn't occurs in normal gameplay**, It is certainly due to warmth / rain protect display script which fire too many times in few seconds.
  
  
**Solution:**
  
Remove or hide bartermenu.swf / craftingmenu.swf
