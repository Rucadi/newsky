**TLDR:**
  
  
Required: CACO or Apothecary
  
Can be used with: **Any** mod overhauling vanilla potion models using vanilla mesh paths.
  
Install: Each is a single ESPFE plugin, so just install it normally and it won't count towards your plugin limit.
  
  
Suggested Load Order:
  
 0. [If using APS, put its plugin above your alchemy mod.]
  
1. CACO or Apothecary.
  
2. Other CACO or Apothecary Patches.
  
3. Potion Remodel Patch from this mod page.
  
  
  
  
**For Those Interested in an Explanation:**
  
  
Vanilla-based potion remodels (mods that replace vanilla meshes directly, instead of via plugin/new pathing) are partly incompatible with CACO/Apothecary. Yes, it's true, even though they are just loose vanilla mesh replacers.
  
  
The issues arise because CACO and Apothecary change potion model pathing to non-vanilla for some potions, and also add new potions that don't use vanilla models.
  
  
As a result, no matter how you sort your plugins, if you use either of these alchemy mods with vanilla-based remodels, you will get a **mix** of meshes in-game. 
  
**Example:**
  
![](https://staticdelivery.nexusmods.com/mods/1704/images/58655/58655-1679276013-960531643.png)﻿
  
  
What these patches do is copy over any potion records from CACO or Apothecary that use non-vanilla meshes, and revert or alter the mesh path to a matching vanilla one instead. As such, the patched potions will be "fixed," inheriting the remodel's meshes in-game while also retaining all of the alchemy mod's changes to potion names/effects/etc.
  
  
**Patched Result:**
  
![](https://staticdelivery.nexusmods.com/mods/1704/images/58655/58655-1679276032-828659905.png)﻿
  
  
  
**There are other *slight* issues for CACO****users that I addressed as well.**
  
  
**Firstly:**
  
Even when CACO reuses vanilla meshes, it often adds alternate texture paths, causing visual bugs in-game if you are using a unique mesh replacer.
  
  
**Secondly:**
  
CACO removes potion bottle size progression from the game. This is a design choice on CACO's part, not a bug. But this design is meant for use with vanilla potion meshes, which CACO adds diversity to using its alternate textures and some new vanilla-style meshes. For users of potion remodels and the patch provided on this page, this feature ends up being a hindrance instead, diminishing the amount of potion models users will readily observe in-game.
