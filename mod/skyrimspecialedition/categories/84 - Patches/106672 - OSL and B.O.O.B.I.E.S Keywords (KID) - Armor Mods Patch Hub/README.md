**DESCRIPTION**
  
  
This mod distributes customized BakaFactory's**SL Aroused Keywords** and **B.O.O.B.I.E.S Keywords** via Keyword Item Distributor (KID) for various armor mods as listed (400+). This means it does not use an ESP/ESL, is light-weight and can easily be changed, updated and deinstalled.
  
  
Some of the listed mods have already been assigned keywords through B.O.O.B.I.E.S (e.g. TAWOBA, TEWOBA, Mysterious Knight etc.). Focus lies on SLA keywords.
  
Can also be used as framework. 
  
  
**INFORMATION ABOUT SLA KEYWORDS**
  
  
Each piece of armor was carefully checked for its external characteristics and assigned one or more keywords according to the Baka Keyword manual. A piece of armor that occupies slot 32 was assigned only one of 8 possible keywords (not considering supportive keywords). I don't use generalizations based on armor slot or equipment name. **Every keyword has been hand placed**.

**SOS patch**
  
SpoilerThis KID patch prevents [Schlongs of Skyrim](https://www.loverslab.com/files/file/5355-schlongs-of-skyrim-se/) to assign slot 52 to supported armors that occupy slot 32**\***. Although this is a feature of SOS rather than a bug, it is perceived as one by many. 
  
The SOS patch is recommended to all users that don't use schlongs on supported outfit wearing characters. Schlong users should refrain from installing this as the distribution of slot 52 by those mods is built-in support to prevent clipping. Consequently, characters that do not have a schlong do not need the built-in support (male NPCs probably still do).
  
  
Keywords:
  
> ﻿SOS\_Revealing - to any item that is slot 32 that does not cover genital area (e.g. bra)
>   
> SOS\_Underwear - to any item that is not slot 32 and covers genital area (e.g. pants)

  
This optional patch does not require the main part of this mod (SLA and BOOBIES keywords) to be installed.
  
  

  
* Requires: [SOS](https://www.loverslab.com/files/file/5355-schlongs-of-skyrim-se/)

  
**⚠️Note 1:**
  
**\***: Outfits that cover upper body and also genital area (e.g. [leotard](https://en.wikipedia.org/wiki/Leotard)﻿) will still gain slot 52.

**Licenses - Player Oppression patch**
  
SpoilerThis KID patch distributes keywords to make the supported armor mods compatible with [Licenses - Player Oppression](https://www.nexusmods.com/skyrimspecialedition/mods/110418). This means that the player will no longer be sanctioned if the correct license is possessed.
  
  
Distributed keywords[\*](https://www.nexusmods.com/skyrimspecialedition/articles/6374) are:
  
> **BM\_LicensesBikiniItem** - marks item as Bikini
>   
> **BM\_LicensesIgnoreItem** - marks item to be ignored

  
Please note that not all supported armor mods receive the corresponding keyword, because not all of them fulfill the requirement as “bikini armor”. Listed as such are generally armors that are revealing. The decision as to whether an armor is considered revealing was made very generously. Depending on your LPO settings you don't have to use this patch, since you can set in LPO MCM what keywords are considered bikini items; e.g. choosing all SLA keywords would make licenses more restrictive, different to this LPO patch which will cover all or most armor parts.
  
  
Few items will gain "**BM\_LicensesIgnoreItem**" keyword (those that are generally not considered armor at all, e.g. [Fur Pelt Cloaks](https://www.nexusmods.com/skyrimspecialedition/mods/120726) or [Nirn Necessities](https://www.nexusmods.com/skyrimspecialedition/mods/112481)). "**BM\_LicensesBikiniItem**" is mostly used for all revealing armors.
  
  
Unlike the main part of this mod, this patch distributes exclusively by ESP name.
  
  
This optional patch does not require the main part of this mod (SLA and BOOBIES keywords) to be installed.
  
  

  
* Requires: [Licenses - Player Oppression](https://www.nexusmods.com/skyrimspecialedition/mods/110418) or [LL-release](https://www.loverslab.com/files/file/29357-licenses-player-oppression/)﻿﻿﻿

Makes it easier to distinguish bikini items and avoid unnecessary penalties for accidentally violating a license.

Requires: Licenses v. 1.25.1.2+﻿
