> **Next-Gen Decapitations aims primarily to fix all known crashes related to decapitations** to date, including those caused by the use of **RaceMenu overlays** or **specific NPC appearance overhauls (FaceGen)**.
>   
>   
> That said, it also introduces **new features to enhance the gameplay experience**, such as NPC helmets being ejected, a customizable spinning effect when the head flies off, and many other features detailed further below.
>   
>   
> Although dependent on SKSE, **the mod is fully standalone** and can be **configured** via an .ini file.

Fully configurable through an .ini file, allowing users to adjust features like head spin mechanics, helmet ejection probabilities and more to suit their preferences.
  
  
Below is **the list of parameters** you can freely modify in the **NextGenDecapitations.ini** file.
  
Spoiler[General]
  
*# This parameter determines the verbosity level used by the mod for logging and communication.*
  
*# Levels of verbosity:*
  
*# 0: Only errors will be logged.*
  
*# 1: Standard information will be logged.*
  
*# 2: Advanced logging will be enabled.*
  
**iVerboseMode = 1**
  
  
[Impulse]
  
*# Determines the impulse factor applied to the severed head.*
  
**fHeadImpulseFactor = 2.8**
  
  
*# Determines the impulse factor applied to the equipment attached to the severed head.*
  
**fHeadEquipmentImpulseFactor = 3.1**
  
  
*# Determines the spin factor applied to the severed head.*
  
**fHeadSpinFactor = 1.2**
  
  
*# Determines the spin factor applied to the equipment attached to the severed head.*
  
**fHeadEquipmentSpinFactor = 0.25**
  
  
[Equipment]
  
*# Allows NPC inventory access during a contextual action on their head. (1 enable, 0 disable).*
  
**bCanBeActivated = 1**
  
  
*# Replicate the NPC's head equipment onto the severed head (1 enable, 0 disable).*
  
**bReplicateHeadEquipment = 1**
  
  
*# Chance (percentage) that clothing equipment on the head will be ejected upon decapitation.*
  
**fClothingEquipmentEjectionChance = 85.0**
  
  
*# Chance (percentage) that light armor equipment on the head will be ejected upon decapitation.*
  
**fLightEquipmentEjectionChance = 70.0**
  
  
*# Chance (percentage) that heavy armor equipment on the head will be ejected upon decapitation.*
  
**fHeavyEquipmentEjectionChance = 40.0**
  
  
[Nodes]
  
*# Z-axis position of the "NPC COM [COM ]" node for male characters (adjust for proper alignment).*
  
**fCOMNodeZPositionMale = 65.0**
  
  
*# Z-axis position of the "NPC COM [COM ]" node for female characters (adjust for proper alignment).*
  
**fCOMNodeZPositionFemale = 65.0**
  
  
[Misc]
  
*# List of EditorFormIDs for races that should not be affected by the Next-Gen Decapitations algorithm (separated by commas).*
  
**sExcludedRaces = DremoraRace,DLC2DremoraRace**
  
  
*# Specifies substrings that, if found in the armor addon model path, will prevent the armor piece from being unequipped (case-insensitive, separated by commas).*
  
**sExcludedArmorModels = wig,hair**

This mod **requires [Skyrim Script Extender (SKSE64)](https://www.nexusmods.com/skyrimspecialedition/mods/30379) to be installed** (any version) and [Address Library for SKSE Plugins](https://www.nexusmods.com/skyrimspecialedition/mods/32444) in order to function.
  
  
For VR users, ensure you have the latest version of [VR Address Library for SKSEVR](https://www.nexusmods.com/skyrimspecialedition/mods/58101) installed for proper functionality.
  
  
While [Death Drop Overhaul](https://www.nexusmods.com/skyrimspecialedition/mods/151590) is technically optional, it is **required for the helmet ejection feature to function properly**, as it handles inventory management without relying on scripts.

The mod can be installed like any other mod via a mod manager or manually.
  
  
**Manual Installation :**
  
Spoiler1. Download the latest version of the mod.
  
2. Open the archive using [7zip](https://www.7-zip.org) or [Winrar](https://www.win-rar.com).
  
3. Place the **entire contents** of the archive into the **"Data" folder** of your game.
  
(Default: C:\Program Files\Steam\Steamapps\common\Skyrim Special Edition\Data)
  
4. Activate "Next-Gen Decapitations.esp" from the game's mod manager (or any other mod manager).
  
5. Launch Skyrim and envoy!

಄ Can the mod cause slowdowns?
  
No slowdowns are expected. However, there might be a slight delay of one or two frames during decapitation as the game performs the necessary calculations for rendering the 3D model of the player's or NPC's head.
  
  
಄ Is the mod compatible with [XYZ]?
  
Yes, Next-Gen Decapitations is fully compatible with all other mods (at least to my knowledge).
  
  
಄ Do I need to make any adjustments to other mods I'm already using (e.g., VioLens, etc.)?
  
No, Next-Gen Decapitations makes fixes at the source code level, so there's no need to adjust any other mods you're using.
  
  
಄ I am experiencing a crash or a specific issue:
  
In that case, please refer to the instructions provided in the pinned message in the comments section.
  
  
಄ What's the difference between Next-Gen Decapitations and mods like Simple Beheading or the Dismembering Framework?
  
**Next-Gen Decapitations focuses entirely on fixing and improving the game's decapitation mechanics**, resolving crashes and glitches associated with the vanilla system. [Simple Beheading](https://www.nexusmods.com/skyrimspecialedition/mods/117172) and the [Dismembering Framework](https://www.nexusmods.com/skyrimspecialedition/mods/126203), on the other hand, are mods that enable decapitations (and dismemberment in DF's case) but still rely on the game's original decapitation function, which Next-Gen Decapitations enhances. **Although Next-Gen Decapitations can be used on its own, it complements the mod you're already using** (whether it's Simple Beheading or the Dismembering Framework).
