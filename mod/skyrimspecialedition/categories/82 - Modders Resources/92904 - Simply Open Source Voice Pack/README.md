# Simply Open Source Voice Pack
- Author: megaman2k
- Game: skyrimspecialedition
- Mod Page: https://www.nexusmods.com/skyrimspecialedition/mods/92904


This plugin adds new voicetypes for use with followers and NPCs.
  
  
**For users**
  
*If a mod is pointing you here*, just download and install, making sure all plugins are enabled.
  
  
*If you want to make an NPC use one of these voices*, please see the article on adding a voice to an NPC with xEdit.
  
  
*If you want to start making your own content with these voices*, please see dialogue tutorials such as the [CK wiki tutorial](https://ck.uesp.net/w/index.php?title=Bethesda_Tutorial_Dialogue), [Darkfox127's video](https://www.youtube.com/watch?v=XQMkVcekHUk), or [CadenBlackGaming's video series](https://www.youtube.com/watch?v=4jpMZ0y7CHA). Use 11labs only to generate the audio files.
  
  
Note the below support for follower frameworks:
  
  

  
* **Nether's Follower Framework** - Compatible.
**Extensible Follower Framework** - *Requires patch provided in the FOMOD.*
  
* **Amazing Follower Tweaks** - Compatible. There is an extra "I need you to do something"
**Immersive Amazing Follower Tweaks** - Compatible.
  
* **Simple Follower Extension** - Compatible.
**Swiftly Order Squad** - Compatible.
  
* **UFO Ultimate Follower Overhaul** - *Not* compatible.
**Multiple Followers Framework** - *Without* the MFF "Dialogue" patch, these voices will use the vanilla follower framework. Otherwise, the you may try the experimental MFF patch (*NOT* included in the FOMOD; see optional files).
  
* **Busy Follower Framework** - Try the experimental BFF patch (*NOT*included in the FOMOD; see optional files).

  
**For mod authors**
  
The goal is to provide the basics of follower dialogue, so that *you*, the mod author, can get a head start with custom, AI-voiced followers *without* wasting time on vanilla follower dialogue. The best part? These voices can work with most follower frameworks! Refer to details per voice.
  
  
**Note:** I *strongly* recommend you use these voicetypes as is instead of duplicating the files to make a custom framework. There is no need to replace the vanilla follower functions (recruit, dismiss, wait, follow, do favor, trade) with yet another standalone, custom framework when these voicetypes already integrate with mature follower frameworks. You can, at any time, pull a follower out of the framework temporarily by dismissing them (see my scripts) and timing them out from the PotentialFollowerFaction. If you have questions about this, or want me to make changes to make this more agreeable to you, feel free to send me a message.
  
  
Unless otherwise specified, these voices support the following vanilla features:
  

  
* All SharedInfos from the DialogueGeneric quest.
Marriage dialogue.
  
* Hearthfires adoption support.
Vendor lines.
  
* Innkeep rumor lines (all *3* of them).

**Integration (for mod authors)**
  
  

  
* To disable whispered lines when the player is sneaking, add your follower to the faction [font=Courier New]VP\_FACT\_NoWhisper[/font].
To disable unique lines when the player enters and exits combat, add your follower to the faction [font=Courier New]VP\_FACT\_NoCombatChangeDialogue[/font].
  
* To disable the default/vanilla vendor lines and go back to previous ones, add your follower to the faction [font=Courier New]VP\_FACT\_NoDefaultServiceDialogue[/font].
If you want custom vendor lines, be sure your quest with new lines has priority greater than 50.
  

  
Old Notes
  
Spoiler
  
To enable spoken NFF dialogue lines, add your follower to [font=Courier New]nwsFF\_VoiceFac[/font] with rank 2. See the demo follower scripts for doing this patch-free. **Note that I do this automatically upon recruitment starting from version 4.3.0.**
  
  
To migrate to the latest version from before 4.0.0:

  
* Delete any usage of [font=Courier New]VP\_FACT\_FixedMarriageDialogue[/font]. This workaround for fixing marriage dialogue is no longer necessary.
If you had updated to reference [font=Courier New]SOSVoices.esl[/font] as a master, open your plugin(s) in zEdit and directly change the master file name from [font=Courier New]SOSVoices.esl[/font] to [font=Courier New]SOSVoices.esm[/font].
  
* If you added/modified lines in the *old* .esm, those have been relocated to one of the additional .esl files.
Clean your masters.
